# VEX AI R&D Research Plan

## Main Question
How should this team structure its early software R&D so the first VEX AI codebase is aligned with the 2025-2026 VAIRC game and the official VEX software model?

## Subtopics
1. Current official game format and scoring model for VAIRC Push Back
2. Official software and sensor capabilities that matter for autonomy
3. Practical repo and code architecture for early team development

## Expected Outputs
- A short list of current-season constraints that shape software design
- A short list of official APIs and systems to target first
- A starter repo layout with a portable decision prototype that can later be ported into VEXcode

## Synthesis Approach
Use current official VEX / RECF sources to define the game and APIs, then turn those constraints into:
- a software roadmap
- a repo scaffold
- a small autonomy-planning prototype
